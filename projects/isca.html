ISCA

Software Engineering Management and Evolution and Software Dependability project

ISCA was developed as a re-engineering project for the ASATDD (Automatic Self-Admitted Technical Debt Detection) tool as part of the Software Dependability and Software Engineering Management and Evolution courses.

The ASATDD tool performs a textual analysis of commit messages to identify and highlight commits that may have introduced self-admitted technical debt as explicitly acknowledged by the developer. The goal of ISCA was to develop a Conversational Agent capable of detecting Self-Admitted Technical Debt within commit messages of a given repository. This Conversational Agent was designed to remove platform constraints, making the tool as accessible as possible for all users. Additionally, the project aimed to deliver a practical tool for industry professionals.

Old system vs new system

A Self-Admitted Technical Debt occurs when a developer acknowledges that they have published incomplete code, code that requires refactoring, introduces bugs, or was implemented as a temporary workaround to ensure system functionality—while noting that future modifications will be necessary. The ASATDD plugin is designed to detect and flag such cases through commit message analysis.

The project was divided into two main phases:

First, we analyzed the features and functionalities of the existing ASATDD plugin.

In the second phase, we focused on developing new components, implementing Change Requests (CRs), and building the Conversational Agent. The key Change Requests identified were:

  • Re-engineering the module responsible for detecting Self-Admitted Technical Debt in ASATDD to make it compatible with the Conversational Agent.
  • Developing a Conversational Agent that utilizes the functionalities of the re-engineered module.
  • Enhancing the classifier by improving the training process.
  • Adding and modifying exception handling to manage potential execution errors in the system.

Traceability matrix of the new system after the execution of Change Requests

The project was developed using Java for the re-engineering of ASATDD, which was originally written in Java, and Python for the Conversational Agent, which was designed to be integrated with Slack.
